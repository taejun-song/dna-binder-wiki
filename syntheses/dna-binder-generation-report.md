# DNA Binder Generation and Evaluation Report

End-to-end report on de novo DNA-binding protein design using [[rfdiffusion3]], comparing our generated binders against Baker Lab reference designs across 9 genome editing targets.

## 1. Objective

Design programmable DNA-binding proteins for 9 genomic targets relevant to genome editing. Evaluate designs using RoseTTAFold3 (RF3) complex structure prediction with pTM and ipTM as primary metrics, following the evaluation criteria from the [[rfdiffusion3-paper]].

## 2. Evaluation Criteria (from RFD3 paper)

The RFD3 paper establishes these thresholds for in silico success:

- **pTM ≥ 0.8** — global fold confidence of the designed protein
- **ipTM ≥ 0.7** — interface confidence for the protein-DNA complex (analogous to inter-chain pAE ≤ 1.5 used with AF3)
- **DNA-aligned RMSD < 5 Å** — structural accuracy of the DNA conformation

We use pTM and ipTM from RF3 as our primary filters. Designs passing both thresholds are considered high-confidence binders.

## 3. Data Sources

### 3.1 Baker Lab Reference Designs

Source: `raw/data/RFD3 binders for Genome Editing Assay -BakerLab.csv`

- **92 pre-designed candidates** across 9 DNA targets
- Generated by Baker Lab using [[rfdiffusion3]] with target-specific DNA conformations
- Includes both initial designs and specblock re-scaffolded variants
- No structural metrics provided in the CSV — we scored all candidates with RF3

### 3.2 Our Generated Designs

Generated using [[rfdiffusion3]] via rc-foundry on an NVIDIA H200 NVL (144GB VRAM).

- **Gen2 batch**: 952 structures across 9 targets using 1BNA (Dickerson dodecamer) as DNA template
- **Overnight batch**: 2,400+ structures generated autonomously, scored continuously
- DNA template: PDB 1BNA (CGCGAATTCGCG, ideal B-DNA), applied to all targets via contig specification

## 4. Methodology

### 4.1 Pipeline Architecture

```
CSV / RFD3 output → ESMFold (monomer) → sequence analysis
                  → RF3 (protein-DNA complex) → pTM + ipTM scoring
                  → redundancy clustering → module identification
```

### 4.2 Baker Lab Candidate Evaluation (Round 1)

1. Loaded 92 candidates from Baker Lab CSV
2. Scored all 92 with RF3 for pTM and ipTM against their respective DNA target sequences
3. 9 parallel RF3 processes on H200, one per target

### 4.3 New Binder Generation (Gen2 + Overnight)

1. Downloaded 1BNA.pdb from RCSB as B-DNA structural template
2. Created RFD3 JSON configs for each target with contigs mapping to 1BNA chains A (residues 1-12) and B (residues 13-24)
3. Ran RFD3 with `n_batches=25-50` per target, multiple waves in parallel
4. Extracted protein sequences from output CIF.gz files
5. Scored protein-DNA complexes with RF3
6. Overnight autonomous runner cycled generation → scoring continuously

### 4.4 Overnight Runner (Autoresearch-inspired)

Inspired by [[autoresearch]], we built an autonomous pipeline that:

- Monitors GPU VRAM and launches RFD3 batches when capacity is available
- Scores completed batches with RF3 immediately
- Logs all results to TSV with timestamps
- Reports new best ipTM per target
- Cycles through all 9 targets round-robin
- Runs unattended overnight (~8 hours)

### 5.4 Sequence Analysis of Passing Designs

All 6 Baker Lab passing designs are specblock variants (re-scaffolded), confirming that the specblock protocol significantly improves binding interface quality. Key sequence features of passing designs:

- **Length range**: 125–150 residues
- **High Arg/Lys content**: all contain DNA-reading motifs (WFQ/WFR/WFN triplets characteristic of homeodomain recognition helices)
- **Conserved HTH motifs**: multiple designs contain helix-turn-helix patterns with QVRNWF or QIRNWF recognition helices

## 6. Analysis

### 6.1 Why Baker Lab Designs Outperform

1. **Target-specific DNA structures**: Baker Lab used AF3-predicted DNA conformations for each target sequence. We used a generic 1BNA template (CGCGAATTCGCG) for all targets. DNA shape varies by sequence — the 1BNA geometry may not match the groove widths and curvatures of TGAGGAGAGGAG or CAGCAGCAGCAG.

2. **Specblock re-scaffolding**: Baker Lab's passing designs are all `_specblock_` variants where the major-groove recognition interface was fixed and the surrounding scaffold was regenerated. This protocol optimizes the interface while maintaining the DNA-contacting residues. Our designs lack this refinement step.

3. **Parameter tuning**: Baker Lab likely optimized RFD3 parameters (ori_token, H-bond conditioning, RASA constraints) per target. We used default parameters for all targets.

### 6.2 Our OCT4pt1 Success

Our one passing design (OCT4pt1_18_model_0, ipTM=0.715) surpasses all 8 Baker Lab OCT4pt1 candidates (best was 0.646). This may be because:

- The 1BNA template happens to have compatible geometry for the 10bp GGTGAAATGA sequence
- RFD3 found a novel fold that the Baker Lab parameter set didn't explore
- This is a shorter target (10bp vs 12bp), and our contig spec (A1-10, B13-22) may produce better fits

### 6.3 Non-Passing Targets

CAG, HSTELO, OCT4pt2, TATA, and Dux4grna2 have no passing designs from either source. These targets likely need:

- Target-specific DNA structure modeling (not generic 1BNA)
- Specblock re-scaffolding of the best initial designs
- Longer protein lengths or different contig specifications
- H-bond and RASA conditioning specific to each DNA sequence

## 7. Recommendations

### 7.1 Immediate (improve scores for current designs)

1. **Apply specblock re-scaffolding** to the top 5 designs per target from our generation — fix the recognition interface and re-diffuse the scaffold
2. **Generate target-specific DNA structures** using x3dna or AF3 for each of the 9 target sequences instead of using 1BNA
3. **Score remaining overnight designs** — the runner is still generating and may find better candidates

### 7.2 Short-term (autoresearch optimization)

4. **Implement the [[autoresearch-for-binder-design]] loop** — sweep RFD3 parameters (protein_length, ori_token, H-bond constraints) per target with ipTM as the optimization metric
5. **Focus on near-passing targets** (OCT4pt2 at 0.685, HSTELO at 0.654) where small parameter improvements could push above threshold

### 7.3 Longer-term (experimental validation)

6. **Prioritize HD and NFKB designs for experimental testing** — these have the highest ipTM scores and multiple independent passing designs
7. **Test OCT4pt1_18_model_0** — our novel design that surpasses Baker Lab for this target
8. **Design cross-target experiments** to test whether the WFQ/WFR recognition helix motifs found across multiple passing designs constitute a transferable DNA-recognition module
